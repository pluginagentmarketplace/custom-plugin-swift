# Career Guidance

Get professional insights, career path recommendations, and guidance for long-term career planning and development.

## Command Usage

```
/career-guidance [goal] [optional: current-role]
```

## Career Paths

### Technical Track (Individual Contributor)
```
Junior Engineer → Mid-Level Engineer → Senior Engineer →
Staff Engineer → Principal Engineer
```

### Management Track
```
Individual Contributor → Team Lead → Manager →
Senior Manager → Director → VP
```

### Specialist Tracks
```
Full-Stack → Specialization (DevOps, Security, AI/ML, etc.)
```

### Hybrid Tracks
```
Engineer ↔ Technical Lead ↔ Engineering Manager
(Flexibility between technical and leadership)
```

## Career Advancement Guidance

### Junior Developer (0-2 years)
**Focus Areas:**
- Master a primary language/framework
- Build solid fundamentals
- Learn company processes
- Get comfortable with tools

**Growth Goals:**
- Complete assigned tasks independently
- Write maintainable code
- Participate in code reviews
- Contribute to team projects

**Timeline:** 1-2 years to mid-level

### Mid-Level Developer (2-5 years)
**Focus Areas:**
- Deep expertise in your domain
- System design understanding
- Code architecture
- Mentoring juniors

**Growth Goals:**
- Lead small projects
- Make design decisions
- Improve performance
- Support team growth

**Timeline:** 2-3 years to senior or specialist

### Senior Developer (5-10 years)
**Focus Areas:**
- Large system design
- Technical strategy
- Team leadership
- Career planning

**Growth Goals:**
- Design major systems
- Define standards
- Mentor multiple people
- Drive innovation

**Options:**
- Continue technical track → Staff/Principal
- Move to management → Engineering Manager
- Become specialist → Expert deep dive

### Leadership Roles (10+ years)

**Engineering Manager:**
- Team management
- Hiring & development
- Performance reviews
- Resource planning

**Staff/Principal Engineer:**
- Technical authority
- Cross-team influence
- Architecture decisions
- Strategic planning

## Career Decision Framework

### Should I Go Technical or Management?

**Choose Technical If:**
- ✅ Love building and coding
- ✅ Enjoy deep technical problems
- ✅ Prefer hands-on work
- ✅ Want to stay close to code
- ✅ Less interested in people management

**Choose Management If:**
- ✅ Enjoy developing people
- ✅ Like strategic planning
- ✅ Want broader organizational impact
- ✅ Interested in business aspects
- ✅ Good at communication and delegation

**Try Hybrid If:**
- ✅ Want flexibility between both paths
- ✅ Technical lead or staff engineer roles
- ✅ Mentor while still coding
- ✅ Flexible role definition

## Specialization Paths

### After 3-5 Years, Consider:

**DevOps/Infrastructure:**
- Container orchestration mastery
- Cloud expertise
- Automation focus
- Career: DevOps Engineer → SRE → Infrastructure Architect

**Security:**
- Security architecture
- Penetration testing
- Compliance expertise
- Career: Security Engineer → Security Architect → CISO

**AI/Machine Learning:**
- Deep learning expertise
- Model optimization
- MLOps focus
- Career: ML Engineer → Research Engineer → ML Architect

**Full-Stack:**
- Broad technology knowledge
- End-to-end system ownership
- Product understanding
- Career: Full-Stack → Tech Lead → CTO

**Product Engineering:**
- Product understanding
- Customer focus
- Feature development
- Career: Product Engineer → Engineering Manager → Head of Product

## Common Career Transitions

### Coding → Management
- Takes 2-3 years to transition
- Learn management skills early
- Mentor while still coding
- Gradual responsibility shift
- Some companies have "tech lead" role

### Web Development → DevOps
- Learn infrastructure fundamentals
- Master containerization
- Study cloud platforms
- Practice CI/CD
- Timeline: 12-18 months

### Backend → Full-Stack
- Learn frontend frameworks
- Build end-to-end systems
- Understand user perspective
- Project ownership increases
- Timeline: 6-12 months

### Bootcamp → Industry
- Build portfolio projects
- Contribute to open source
- Network aggressively
- Continuous learning
- Timeline: 6-24 months to get hired

## Work-Life Balance

### Career Stages & Balance
```
Junior: High learning, work-life balance possible
Mid: Growth opportunities, some pressure
Senior: Flexibility increases, strategic focus
Manager: Different trade-offs, team responsibility
```

### Healthy Career Management
- Set boundaries
- Continuous learning
- Networking time
- Side projects (optional)
- Personal projects
- Time with family

